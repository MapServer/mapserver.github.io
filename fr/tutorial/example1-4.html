Exemple 1.4¶

Etiqueter la carte¶

Nous pouvons aussi rajouter des étiquettes à notre carte...

MapServer a un système de gestion des étiquettes très flexible. Il supporte en natif les polices bitmap comme truetype. LA mise à l’échelle des polices est supportée pour les polices truetype. Les angles et les placements des étiquettes peuvent être personnalisés... Si vous prenez le temps d’apprendre les nombreux paramètres impliqués dans la création de bonnes étiquettes, vous serez récompensés avec des cartes informatives et esthétiquement jolies.

C’est ce à quoi le mapfile ressemble: Example1-4.map.

Structure d’un MapFile¶

La structure du fichier .map, par objets, ressemble à ceci:

                             MAP
 (states_poly) LAYER----------|---------LAYER (states_line)
(land) CLASS-----|-CLASS (water)         |-CLASS
  STYLE-|-LABEL     |-STYLE                 |-STYLE</pre>

Mots-clés et objets de LABEL¶

Ici, nous introduisons quelques paramètres en plus avec l’objet LABEL:

FONTSET

Ici nous spécifionsle chemin complet vers notre fichier de liste de polices truetype (ou jeu de polices). Ce fichier liste chacune des polices disponibles. Regardez le fichier directement et la documentation du jeu de polices MapServer pour plus d’informations. FONTSET est une paramètre de l’objet MAP.

LABELITEM

Cela indique quelle donnée attributaire utiliser pour l’étiquetage, dans ce cas “STATE”. LABELITEM est un paramètre de l’objet LAYER.

LABEL

Marque le début de l’objet LABEL. l’objet LABEL peut être utilisé à l’intérieur d’autres objets (comme l’objet de barre d’échelle SCALEBAR).

COLOR

A l’intérieur de l’objet LABEL, COLOR spécifie la couleur de l’étiquette texte.

SHADOWCOLOR - deprecated

Cela spécifie la couleur de l’ombre de l’étiquette texte.

Note

Depuis MapServer 6, le LABEL STYLE avec GEOMTRANSFORM LABELPOLY doit être utilisé à la place. Les fichiers du tutoriel n’ont pas été mis à jour.

SHADOWSIZE - deprecated

Spécifie la taille de l’ombre. La valeur correspond aux décalages de X et Y en pixels. Ainsi, “2 2” signifie deux pixels de large par deux pixels de haut.

Note

Depuis MapServer 6, le LABEL STYLE avec GEOMTRANSFORM LABELPOLY doit être utilisé à la place. Les fichiers du tutoriel n’ont pas été mis à jour.

TYPE

A l’intérieur de l’objet LABEL, le TYPE spécifie quelle type de police utiliser. Nous avons le choix de TRUETYPE ou BITMAP (les polices intégrées par défaut). Nous choisissons TRUETYPE.

FONT

Si vous indiquez le TYPE comme TRUETYPE, vous devrez spécifier quelle police utilisée. La valeur ici est “alias” dans le fichier de liste des polices.

SIZE

Si on utilise des polices truetype, la valeur de SIZE est en pixels. Si ce sont des polices bitmap, vous pouvez indiquer quleque chose “small” ou “large”.

ANTIALIAS

Cela active ou désactive l’anticrénelage (antialiasing) truetype. Rappelez-vous que la valeur n’est pas on ouoff mais TRUE ou FALSE.

POSITION

Où positionner le texte des étiquettes en relation avec les points des étiquettes. La valeur est une combinaison de positions verticale et horizontale. Vous avez les choix suivants pour l’alignement vertical: C pour centre (Center), U pour Haut(Upper), et L pour bas (lower). Pour l’alignement horizontal, vous avez les choix suivants: C pour centre (Center), L pour gauche (left) et R pour droite (right). Ainsi, pour aligner l’étiquette au centre de l’identifiant d’étiquette, vous utiliserez la valeur “CC” (centre-centre). Ou si vous vous voulez être à la partie en bas à gauche de l’identifiant, vous utiliserez LL. Une autre manière est de laisser MapServer décider la meilleure position pour vos étiquettes. Pour cela, vous devrez utilisez la valeur “AUTO”.

PARTIALS
Tells MapServer whether to generate incomplete label texts or not. The default here is not to generate fragments of a label text. The value is either TRUE or FALSE.
MINDISTANCE

C’est la distance minimum en pixels entre les étiquettes dupliquées. Regardez ce qu’il se passe si vous augmentez ou diminuez cette valeur.

BUFFER

L’alignement (en pixels) pour chaque étiquette. Cela est utilisé pour amélioré la lisibilité. Un tampon ou BUFFER de 4 pixels signifie qu’aucune étiquette ne sera dessinée à moins de 4 pixels les uns des autres. A nouveau, faites des changements pour voir comment ça marche.

Vous pouvez aussi créer des étiquettes séparées d’une couche POLYGON. Vous faîtes cela avec le TYPE de données ANNOTATION. Regardez le prochain exemple de fichier .map pour voir comment implémenter ce type d’étiquettes. Vous noterez que l’objet CLASS à l’intérieur de la couche “label” a un paramètre COLOR à “-1 -1 -1”. Le nombre négatif dit à Mapserver de donner à cette CLASS une couleur transparente (Les identifiants de labels ne se voient pas). Une fois de plus, jouez avec les valeurs pour comprendre comment cela affecte la carte.
