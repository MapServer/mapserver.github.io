Exemple 1.2¶

Carte statique avec deux couches¶

Comme dans le premier exemple, cette image a été générée en pointant la source du tag <img> vers cette URL: http://demo.mapserver.org/cgi-bin/mapserv?map=/osgeo/mapserver/tutorial/htdocs/example1-2.map&layer=states_poly&layer=states_line&mode=map. C’est comme ça que fonctionne la plupart des exemples de cette section.

De vous façon, vous noterez que la carte ici ressemble ici est la même que celle du premier exemple. Oui, mais le mapfile est différent. Regardez.

C’est ce à quoi le mapfile ressemble: Example1-2.map

Structure d’un MapFile¶

La structure d’un mapfile, par objets, ressemble à ceci:

           MAP
      LAYER-|-LAYER
   CLASS-|     |-CLASS
STYLE-|           |-STYLE

Ici nous séparons la couche originale en deux couches. La première couche est toujours une couche polygon mais l’objet STYLE n’a plus une OUTLINECOLOR, comme ceci:

LAYER
  ...
  TYPE         POLYGON
  ...
  CLASS
    NAME       "States"
    STYLE
      COLOR    232 232 232
    END
  END
END

La seconde couche est similaire à la première exceptée que le TYPE est changé à LINE et la couleur COLOR dans le STYLE est changée à la même couleur que la OUTLINECOLOR dans le premier exemple. Cela produit la même image que la première.

LAYER
  ...
  TYPE         LINE
  CLASS
    NAME       "State Boundary"
    STYLE
      COLOR    32 32 32
    END
  END
END

Ainsi, pourquoi le faire de cette manière?¶

Si nous continuons à ajouter des couches par dessus la couche states, le contour sera probablement couvert par ces autres couches. Pour toujours voir les bordures des états après que nous ayions ajouter ces autres couches, nous avons séparés la ligne de contour de la couche states de la couche polygone de la couche states et rajoutées par dessus les autres couches. Il y a un ordre sur comment nous définissons/ajoutons des couches et vous le verrez clairement au fur et à mesure que nous avancerons dans cette section.
