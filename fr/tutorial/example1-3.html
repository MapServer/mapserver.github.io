Structure d’un MapFile¶

C’est ce à quoi le mapfile ressemble: Example1-3.map.

La structure d’un mapfile, par objets, ressemble à ceci:

                            MAP
(states_poly) LAYER----------|---------LAYER (states_line)
               |                        |
(land) CLASS---|---CLASS (water)        |-CLASS
          |         |                      |
    STYLE-|         |-STYLE                |-STYLE

Mots-clés¶

Notre mapfile n’a toujours que deux couches mais la couche polygon a été découpée en deux classes. Jetons un oeil aux paramètres ajoutés:

CLASSITEM

Ce mot-clé est utilisé pour spécifier quel attribut utiliser pour séparer les objets CLASS. Dans cet exemple, cet attribut est “CLASS”. Si vous ouvrez le fichier de données attributaires associés avec le shapefile de la couche, vous verrez qu’il y a une colonne (un attribut) appelé “CLASS”.

Comment savoir quel attribut utiliser? Bien, les enregistrements dans les shapefiles sont stockés dans les fichiers DBF. Vous pouvez les ouvrir dans un logiciel de type comme Calc dans Openoffice.org, ou dans un logiciel SIG bureautique comme QGIS, Thuban, ou ArcView. Si votre donnée est fournie avec une métadonnée (Et cela devrait être le cas!), vous pouvez parcourir le fichier de métadonnée pour avoir les informations attributaires. Vous pouvez aussi utiliser “ogrinfo” pour afficher les attributs de vos shapefiles–revenez à l’exemple example 1.1 (les dernières lignes après “Layer SRS WKT:” montrent les noms et les types des attributs).

EXPRESSION

Pour chaque classe, nous spécifions quelle valeur attributaire utiliser. C’est la forme la plus simple des EXPRESSION. Les EXPRESSION peuvent être plus complexes que ça, permettant d’évaluer des expressions régulières ou logiques. Merci de regarder LES Expressions dans la référence du mapfile pour voir ce que nous pouvons faire avec l’objet EXPRESSION.
