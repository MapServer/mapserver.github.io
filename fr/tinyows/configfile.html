Configurer TinyOWS avec un fichier XML¶

La manière la plus simple de configurer TinyOWS est avec un simple fichier XML appelé tinyows.xml.

Le chemin par défaut est /etc/tinyows.xml. Vous pouvez aussi utiliser You can also use TINYOWS_CONFIG_FILE environment variable to set your own path.

Exemple simple de fichier de configuration¶

Un exemple de fichier config.xml est dans le répertoire de démo:

<tinyows online_resource="http://127.0.0.1/cgi-bin/tinyows"
         schema_dir="/usr/local/tinyows/schema/">

 <pg host="127.0.0.1" user="postgres" password="postgres" dbname="tinyows_demo" port="5432"/>

 <metadata name="TinyOWS Server"
           title="TinyOWS Server - Demo Service" />

 <layer retrievable="1"
        writable="1"
        ns_prefix="tows"
        ns_uri="http://www.mapserver.org/tinyows/"
        name="world"
        title="World Administrative Boundaries" />

 <layer retrievable="1"
        writable="1"
        ns_prefix="tows"
        ns_uri="http://www.mapserver.org/tinyows/"
        name="france"
        title="French Administrative Sub Boundaries (IGN - GeoFLA Departements)" />

</tinyows>

Testez le fichier config.xml¶

A partir du moment où vous avez un fichier config.xml associé à votre service, lancez TinyOWS avec l’option –check pour valider votre fichier de configuration, testez votre connexion à la base de données et listez les couches à utiliser

./tinyows --check
 TinyOWS version:   1.1.0
 FCGI support:      Yes
 Config File Path:  /etc/tinyows.xml (TinyOWS XML)
 PostGIS Version:   2.1.0
 PostGIS dsn:       host=127.0.0.1 user=postgres password=postgres dbname=foo port=5432
 Output Encoding:   UTF-8
 Database Encoding: UTF8
 Schema dir:        /usr/local/share/tinyows/schema/
 Display bbox:      Yes
 Estimated extent:  No
 Check schema:      Yes
 Check valid geoms: No
 Available layers:
  - public.commune (2154) -> tows.commune [RW]
  - public.world (4326) -> tows.world [RW]

Structure d’un fichier config.xml¶

Elément TinyOWS¶

TinyOWS is the root element. He is mandatory, and must contains some system information about the service itself. Some globals service options could also be switched on or off at this level.

Attribut

Requis?

Par défaut

 Description
online_resource

obligatoire

  

URL où le service est situé, c’est à dire http://127.0.0.1/cgi-bin/tinyows
schema_dir

obligatoire

  

Chemin où le répertoire de schéma de TinyOWS est situé, c’est à dire /usr/local/tinyows/schema/
log

optionnel

  

Chemin où TinyOWS loggue les requêtes en entrée, c’est à dire /var/log/tinyows.log. Ce fichier doit avoir des droits en écriture pour l’utilisateur qui a lancé le processus TinyOWS.
log_level

optionnel

 0

Valeur du champ pour indiquer ce qu’on loggue: 1: ERREUR, 2: EVENEMENT, 4: REQUETES HTTP, 8: SQL. e.g: 15 pour tout logguer.
degree_precision

optionnel

 6

Indique le nombre de chiffres après la virgule pour la précision quand les coordonnées sont exprimées en latitude/longitude
meter_precision

optionnel

 0

Indique le nombre de chiffres après la virgule pour la précision quand les coordonnées sont projetées (donc en mètres).
display_bbox

optionnel

 1

“Drapeau” pour indiquer si le rectangle englobant dedoit être généré pour la sortie GetFeature du GML WFS. Il est obligatoire dans la spécification WFS. Mais comme il consomme du temps, il peut être intéressant d’avoir la possibilité de le désactiver.
estimated_extent

optionnel

 0

“Drapeau” pour indiquer si TinyOWS doit utiliser estimated_extent (plus rapide mais un peu moins juste).
check_schema

optionnel

 1

“Drapeau” pour indiquer si la donnée en entrée doit être vérifiée vis à vis du schéma avant d’être exécutée. Attention, la validation de schéma est une part importante du processus de sécurité. Désactiver cet attribut est à vos risques et périls.
check_valid_geom

optionnel

 1

“Drapeau” pour indiquer si la validation des géométries OGC SFS 1.1 doit être effectuée avant d’exécuter la transaction.
encoding

optionnel

 UTF-8

Encodage de sortie. Les autres valeurs par exemple pourrait être ISO-8859-1. Aucune conversion des données n’est effectuée sur les données. Cet attribut est déclaratif.
expose_pk

optionnel

 0

“Drapeau” pour indiquer si TinYOWS devrait exposer la clé primaire dans le schéma (et donc si elle est requise dans la requête transactionnelle)
wfs_default_version

optionnel

  

Chaîne de version permettant d’indiquer la version par défaut de WFS, 1.0.0 ou 1.1.0 par exemple.

Elément Limits¶

L’élément Limits fournit un maximum pour la sortie du serveur. Il peut aider à prévenir une attaque par dénis de service (DDOS), ou empêcher une requête vraiment trop importante de crasher votre serveur. Cet élement est optionnel

Attributs de Limits

Attribut

Requis?

Par défaut

 Description
features

optionnel

  

Utilisé pour définir un nombre maximum de géométries retournées au client XFS lors on d’une requête GetFeature
geobbox

optionnel

  

Etendue géographique du rectangle englobant de l’étendue maximale: Est,Ouest,Nord,Sud

 
<tinyows>
 ...
 <limits features="10000" />
 ...
</tinyows>

optionnel

  

Utilisateur PostgreSQL pour se connecter (par défaut c’est l’utilisateur système utilisé pour faire tourner le serveur)
password

optionnel

  

Mot de passe de connexion PostgreSQL

optionnel

  

Nom de la base de donnée PostGIS (par défaut, le même que celui que l’utilisateur système utilise pour faire tourner le serveur)

optionnel

 5432

Numéro du port PostgreSQL

optionnel

 UTF-8

Encodage de la base de données, comme spécifié dans http://www.postgresql.org/docs/9.0/static/multibyte.html#CHARSET-TABLE

        user="postgres"
        password="postgres"
        dbname="gis_data"

  ...

         email="tinyows-users@lists.maptools.org" />

Couches imbriquées¶

Les entités Layer peuvent être imbriquées, les propriétés dans ce cas peuvent être héritées. Une Layer sans titre est considéré comme une couche ‘virtuelle’

<tinyows>
 ...
     <layer name="root"
            retrievable="1" writable="1"
            ns_prefix="tows"
            ns_uri="http://www.mapserver.org/tinyows/"
            schema="my_db_schema">

          <layer name="foo" title="foo" />
          <layer name="bar" title="bar" />

     </layer>

 ...
</tinyows>
