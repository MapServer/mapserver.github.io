Configurer TinyOWS avec un fichier Mapfile standard¶

Support d’un fichier Mapfile de configuration pour TinyOWS¶

TinyOWS supports as a configuration file a standard MapServer Mapfile. This allow a single file to configure both MapServer and TinyOWS. (e.g could be useful if you use them both, as one for WMS and the other as WFS-T)

TinyOWS ne gère pas tous les paramètres dans un MapFile mais il ignorera, sans erreurs, tous les paramètres supplémentaires qui ne sont pas implémentés dans TinyOWS.

Si vous préférez, nous pouvons configurer TINYOWS en utilisant un fichier XML (Configurer TinyOWS avec un fichier XML).

Pour indiquer où votre Mapfile est situé au binaire TinyOWS. Utilisez la variable d’environnement ‘’TINYOWS_MAPFILE’’ .

Ici un exemple de Mapfile seul

MAP
    NAME "TinyOWS"

    WEB
         METADATA
                    "tinyows_schema_dir" "/usr/local/share/tinyows/schema/"
                    "tinyows_onlineresource" "127.0.0.1/cgi-bin/tinyows.fcgi"
                    "wfs_title" "TinyOWS service provided by a MapFile"
                    "wfs_contact" "foo@bar.net"
         END
    END

    LAYER
            NAME 'France'
            CONNECTIONTYPE postgis
            CONNECTION "host=127.0.0.1 user=postgres password=postgres dbname=tinyows_demo port=5432"
            METADATA
                    'wfs_title' 'France'
                    'wfs_namespace_prefix' 'tows'
                    'wfs_namespace_uri' 'http://www.mapserver.org/tinyows/'
                    'wfs_srs' 'EPSG:27582'
                    'tinyows_table'  'france'
                    'tinyows_writable' '1'
                    'tinyows_retrievable' '1'
            END
            DUMP TRUE
    END
END

Concepts courants et limitations

  • Seule la CONNECTIONTYPE PostGIS est gérée

  • TinyOWS ne supporte pas tous les paramètres WFS disponibles dans un Mapfile. Mais d’un autre côté, vous êtes capable de configurer chaque partie de TinyOWS avec un Mapfile.

  • La valeur de la chaîne de CONNECTION dans chaque couche doit être la même.

  • Le contenu de PROJECTION dans le Mapfile n’est pas parsé, ainsi, vous devez explicitement spécifier wfs_srs.

  • Les paramètres LAYER et FILTER du Mapfile ne sont pas parsés.

  • Les valeurs par défaut sont celles de TinyOWS, même pour les valeurs en commun partagées à la fois par TinyOWS et MapServer.

  • TinyOWS n’utilise pas l’élement DATA du Mapfile, ainsi vous devez utiliser tinyows_table (et tinyows_schema si nécessaire) dans chaque couche.

  • Si DUMP n’est pas défini à TRUE sur la couche, les accès en écriture comme lecture sont désactivés pour la couche.
